# Security remediation — committed Terraform state (issue #1)

The Gen 1 Terraform state, its backup, and the real `terraform.tfvars` were
committed to this **public** repository. The state contained, in plaintext, the
cluster **munge key**, the **slurmdbd database password**, the AWS **account ID**,
and the head node's **public IP**.

This document tracks the remediation. The steps below are split into what has
already been done in-repo and what a maintainer must still do manually, because
the remaining steps are destructive (history rewrite) or touch live
infrastructure (secret rotation).

## Done in-repo (this branch)

- [x] `git rm --cached` the three files so they are no longer tracked. They
  remain on the local disk for whoever is actively using the cluster, but git
  no longer follows them.
- [x] Added `.github/workflows/secrets-guard.yml` — CI fails if any
  `*.tfstate`, `terraform.tfvars`, or private key is ever tracked again.
- [x] Added `scripts/hooks/pre-commit` (+ `scripts/hooks/install.sh`) as a local
  backstop. `.gitignore` only protects *untracked* files; these hooks also catch
  `git add -f` and files staged before an ignore rule.

## Still required from a maintainer (manual)

These cannot be done safely from a feature branch and are left to you.

### 1. Rotate the exposed secrets

Anything that was ever in the committed state must be treated as compromised.

- **Munge key + slurmdbd DB password.** The simplest path for a disposable lab
  cluster is to `terraform destroy` the affected cluster and redeploy — both are
  regenerated by `random_bytes.munge_key` / `random_password.slurmdbd_db` on a
  fresh apply, producing new values. If the cluster must stay up, regenerate the
  munge key on every node and update the slurmdbd password by hand.
- **AWS account ID `942542972736`.** Not a secret on its own, but now publicly
  associated with this lab. No action strictly required; be aware it is exposed.
- **AMI `ami-01ad8723e15011289`, key pair `burstlab-key`, IP `52.33.58.129`.**
  Review whether any should be rotated/retired.

### 2. Purge the files from git history

Removing them from the current commit does **not** remove them from history — the
secrets are still reachable in earlier commits. Because the repo is public, scrub
history and force-push:

```bash
# Using git-filter-repo (recommended; install via brew/pip)
git filter-repo --invert-paths \
  --path terraform/generations/gen1-slurm2205-rocky8/terraform.tfstate \
  --path terraform/generations/gen1-slurm2205-rocky8/terraform.tfstate.backup \
  --path terraform/generations/gen1-slurm2205-rocky8/terraform.tfvars

git push --force --all
git push --force --tags
```

> Force-pushing rewrites public history. Coordinate with anyone who has cloned
> the repo — they will need to re-clone or rebase. This is why it is left as a
> deliberate manual step rather than baked into this branch.

### 3. After history is clean

- Delete this file (its job is done) or keep it as an incident record.
- Close issue #1.
